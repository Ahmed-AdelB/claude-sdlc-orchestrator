# Brainstorm Phase

Begin SDLC Phase 1: Brainstorm

## Instructions

Analyze the user's request and perform comprehensive brainstorming:

1. **Understand the Request**
   - Identify core problem/goal
   - List key requirements
   - Note constraints and assumptions

2. **Explore Solutions**
   - Generate multiple approaches
   - Consider trade-offs
   - Evaluate feasibility

3. **Identify Questions**
   - Clarifying questions for user
   - Technical uncertainties
   - Scope boundaries

4. **Output Format**
   ```markdown
   ## Problem Statement
   [Clear description]

   ## Requirements
   - Functional: [list]
   - Non-functional: [list]

   ## Proposed Approaches
   1. [Approach A] - Pros/Cons
   2. [Approach B] - Pros/Cons

   ## Open Questions
   - [Question 1]
   - [Question 2]

   ## Recommended Approach
   [Selected approach with justification]
   ```

5. **Next Step**: Once approved, proceed to `/document`
